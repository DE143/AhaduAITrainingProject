# Outputs Directory

This directory stores all generated outputs from the credit scoring analysis, including visualizations, reports, and model artifacts.

## Directory Structure

```
outputs/
├── plots/              # Visualization outputs
│   ├── eda/           # Exploratory data analysis plots
│   ├── features/      # Feature engineering visualizations
│   └── models/        # Model performance plots
├── reports/           # Analysis reports
│   ├── data_quality/  # Data quality reports
│   ├── eda/          # EDA summary reports
│   └── model/        # Model evaluation reports
└── models/           # Saved model artifacts
    ├── baseline/     # Baseline models
    ├── production/   # Production-ready models
    └── experiments/  # Experimental models
```

## File Naming Conventions

### Plots
- Format: `{analysis_type}_{feature_name}_{date}.png`
- Example: `eda_age_distribution_20260415.png`

### Reports
- Format: `{report_type}_{date}.html` or `.pdf`
- Example: `data_quality_report_20260415.html`

### Models
- Format: `{model_type}_v{version}_{date}.pkl`
- Example: `xgboost_v1.0_20260415.pkl`

## Usage

All outputs are automatically generated by running the analysis notebooks. To generate outputs:

1. Run `notebooks/eda_analysis.ipynb` for EDA outputs
2. Run `notebooks/feature_engineering.ipynb` for feature outputs
3. Run `notebooks/model_development.ipynb` for model outputs

## Version Control

- ⚠️ Large files (>10MB) should not be committed to Git
- Use `.gitignore` to exclude large model files
- Store production models in cloud storage (S3, Azure Blob, etc.)

## Retention Policy

- **Plots**: Keep for 6 months
- **Reports**: Keep for 2 years
- **Models**: Keep production models indefinitely, experiments for 3 months
